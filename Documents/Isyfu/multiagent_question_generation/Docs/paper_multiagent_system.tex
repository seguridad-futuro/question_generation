\noindent\textbf{Resumen.}
Se presenta un sistema multi-agente basado en LangGraph para la generaci\'on autom\'atica
de preguntas tipo test a partir de documentos legislativos. El sistema emplea cinco agentes
especializados que operan en un patr\'on Map-Reduce: (Z)~fragmentaci\'on coherente de documentos,
(B)~generaci\'on de preguntas con aprendizaje few-shot y 8 t\'ecnicas de distracci\'on documentadas,
(C)~evaluaci\'on dual de calidad mediante criterios especializados y m\'etricas Ragas,
(D)~persistencia con deduplicaci\'on sem\'antica, y (E)~exportaci\'on a PDF profesional.
La arquitectura permite procesamiento paralelo, reintentos autom\'aticos con retroalimentaci\'on
espec\'ifica, y un esquema de base de datos relacional compatible con Supabase.
El sistema ha sido dise\~nado para la preparaci\'on de oposiciones a la Guardia Civil espa\~nola,
procesando actualmente 23 temas legislativos.
\vspace{14pt}

\section{Introducci\'on}
% ============================================================================

La preparaci\'on de oposiciones requiere un volumen masivo de preguntas tipo test de alta
calidad, fundamentadas en legislaci\'on vigente y con distractores plausibles. La creaci\'on
manual de estas preguntas es un proceso costoso, lento y propenso a inconsistencias.

Este documento describe un sistema multi-agente que automatiza el ciclo completo:
desde la ingesta de documentos PDF legislativos hasta la generaci\'on de ex\'amenes en PDF,
pasando por la generaci\'on inteligente de preguntas, validaci\'on de calidad y persistencia
en base de datos.

El sistema se distingue por:
\begin{itemize}[nosep,leftmargin=*]
  \item Arquitectura de 5 agentes con responsabilidades claras
  \item Aprendizaje few-shot con 6 ejemplos reales de ex\'amenes
  \item 8 t\'ecnicas de distracci\'on basadas en an\'alisis de 105 ex\'amenes reales
  \item Validaci\'on dual: criterios especializados + m\'etricas Ragas
  \item Deduplicaci\'on sem\'antica con embeddings de OpenAI
  \item Recuperaci\'on multi-chunk con FAISS para contexto enriquecido
\end{itemize}

\section{Agentes del Sistema}
% ============================================================================

\subsection{Agente Z: Fragmentaci\'on Coherente}

El Agente~Z es responsable de transformar documentos legislativos en bruto en
\textbf{chunks sem\'anticamente coherentes}. Implementado como un \texttt{StateGraph}
con patr\'on ReAct, realiza las siguientes tareas:

\begin{enumerate}[nosep,leftmargin=*]
  \item \textbf{Carga del documento}: Soporta PDF (pypdf), TXT y MD
  \item \textbf{Filtrado de metadatos}: Elimina encabezados, pies de p\'agina,
        emails, URLs y n\'umeros de p\'agina mediante el \texttt{PDFMetadataFilter}
  \item \textbf{An\'alisis estructural}: Identifica art\'iculos, cap\'itulos, secciones
        y t\'itulos del documento legal
  \item \textbf{Fragmentaci\'on}: Genera chunks con tama\~no configurable (por defecto
        1000 caracteres, 200 de solapamiento)
  \item \textbf{Evaluaci\'on de sustantividad}: Calcula un ratio que mide el contenido
        legislativo real vs.\ ruido (\texttt{min\_substantive\_ratio: 0.3})
  \item \textbf{Aprendizaje de patrones}: Registra patrones regex de limpieza
        para reutilizaci\'on entre documentos
\end{enumerate}

Cada chunk generado incluye: \texttt{chunk\_id}, \texttt{content},
\texttt{source\_document}, \texttt{page}, \texttt{token\_count},
\texttt{is\_substantive} y \texttt{metadata}.

\begin{tcolorbox}[colback=agentZ!5, colframe=agentZ, title={\small Modelo de Chunk},
  fonttitle=\bfseries\small, boxrule=0.5pt, arc=3pt]
{\small\ttfamily
class Chunk:\\
\hspace*{1em}chunk\_id: str\\
\hspace*{1em}content: str\\
\hspace*{1em}source\_document: str\\
\hspace*{1em}page: int\\
\hspace*{1em}token\_count: int\\
\hspace*{1em}is\_substantive: bool\\
\hspace*{1em}substantive\_ratio: float\\
\hspace*{1em}related\_chunk\_ids: List[str]
}
\end{tcolorbox}

% ---
\subsection{Agente B: Generaci\'on de Preguntas}

El Agente~B constituye el n\'ucleo generativo del sistema. Implementado como
\texttt{StateGraph} con aprendizaje few-shot, genera preguntas tipo test
de calidad profesional.

\subsubsection{Aprendizaje Few-Shot}
Se incorporan \textbf{6 ejemplos reales} de ex\'amenes oficiales de la Guardia Civil
directamente en el prompt del sistema. Estos ejemplos establecen el formato,
nivel de dificultad y estilo esperado.

\subsubsection{T\'ecnicas de Distracci\'on}
Bas\'andose en el an\'alisis de 105 ex\'amenes reales, se han documentado 8 t\'ecnicas
para crear distractores plausibles (Tabla~\ref{tab:distractors}).

\begin{table}[H]
\centering
\caption{T\'ecnicas de distracci\'on documentadas}
\label{tab:distractors}
\scriptsize
\begin{tabularx}{\columnwidth}{cX}
\toprule
\textbf{\#} & \textbf{T\'ecnica} \\
\midrule
1 & Cambio de conjunci\'on/disyunci\'on (Y $\leftrightarrow$ O) \\
2 & Manipulaci\'on de plazos temporales \\
3 & Eliminaci\'on de excepciones legales \\
4 & Atribuci\'on incorrecta de funciones \\
5 & Confusi\'on de nivel normativo (ley vs.\ reglamento) \\
6 & Falsa limitaci\'on de \'ambito \\
7 & Alteraci\'on de cifras o cantidades \\
8 & Contexto correcto con final modificado \\
\bottomrule
\end{tabularx}
\end{table}

\subsubsection{Recuperaci\'on Multi-Chunk}
Para preguntas que requieren contexto de m\'ultiples art\'iculos, el sistema emplea
el \texttt{ChunkRetrieverService} basado en FAISS:

\begin{enumerate}[nosep,leftmargin=*]
  \item Indexa todos los chunks del documento con embeddings
  \item Para cada chunk primario, recupera los $k$ chunks m\'as similares
        (\texttt{max\_related\_chunks: 3})
  \item Construye un \texttt{EnrichedContext} combinando chunk primario +
        chunks relacionados
  \item Eval\'ua suficiencia del contexto antes de generar
\end{enumerate}

% ---
\subsection{Agente C: Evaluaci\'on de Calidad}

El Agente~C implementa un \textbf{sistema dual de validaci\'on} que combina
criterios especializados para oposiciones con m\'etricas automatizadas.

\subsubsection{Evaluaci\'on Especializada (Primaria)}
Cinco criterios ponderados por criticidad:

\begin{itemize}[nosep,leftmargin=*]
  \item[\textbf{C}] \textbf{Referencias legales expl\'icitas} --- La pregunta debe
        citar art\'iculos o leyes espec\'ificas
  \item[\textbf{C}] \textbf{Precisi\'on t\'ecnica absoluta} --- Sin ambig\"uedades ni
        errores jur\'idicos
  \item[\textbf{A}] \textbf{Distractores plausibles} --- Las opciones incorrectas
        deben ser cre\'ibles
  \item[\textbf{A}] \textbf{Retroalimentaci\'on estructurada} --- El ``tip'' debe
        explicar por qu\'e la respuesta es correcta
  \item[\textbf{M}] \textbf{Formato oficial} --- 4 opciones, sin ambig\"uedad en
        la correcta
\end{itemize}

{\scriptsize (C = Cr\'itico, A = Alto, M = Medio)}

\subsubsection{M\'etricas Ragas (Complementaria)}
\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Faithfulness}: Es la respuesta fiel al contexto? ($0$--$1$)
  \item \textbf{Answer Relevancy}: Es la respuesta relevante a la pregunta? ($0$--$1$)
\end{itemize}

% ---
\subsection{Agente D: Persistencia y Deduplicaci\'on}

El Agente~D gestiona la inserci\'on en base de datos con deduplicaci\'on sem\'antica:

\begin{enumerate}[nosep,leftmargin=*]
  \item Genera embedding de \texttt{question + correct\_answer}
  \item Calcula similitud coseno contra preguntas existentes
  \item Si similitud $> 0.85$: marca como duplicado
  \item Si es \'unica: inserta en SQLite con todos los metadatos
  \item Genera estad\'isticas: total, \'unicas, duplicadas
\end{enumerate}

% ---
\subsection{Agente E: Generaci\'on de PDFs}

Implementado con ReportLab, genera tres formatos de salida:

\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Solo examen}: Preguntas con opciones, sin soluciones
  \item \textbf{Con soluciones}: Incluye tips y explicaciones
  \item \textbf{Hoja de respuestas}: Plantilla de correcci\'on r\'apida
\end{itemize}

Las preguntas se agrupan por tema y se incluyen estad\'isticas de dificultad.

\subsection{main.py --- Pipeline Completo}

Orquesta la ejecuci\'on secuencial de los cinco agentes. Es el punto de entrada
principal para producci\'on.

\begin{tcolorbox}[colback=bglight, colframe=bordercolor, boxrule=0.5pt, arc=2pt]
{\small\ttfamily
python main.py \textbackslash\\
\hspace*{1em}--doc "Tema-18" \textbackslash\\
\hspace*{1em}--questions 5 \textbackslash\\
\hspace*{1em}--topic 101 \textbackslash\\
\hspace*{1em}--academy 1 \textbackslash\\
\hspace*{1em}--skip-agent-c ~~\# opcional
}
\end{tcolorbox}

\textbf{Par\'ametros}:
\begin{itemize}[nosep,leftmargin=*,font=\small\ttfamily]
  \item[{-}{-}doc] Patr\'on de filtrado de documentos
  \item[{-}{-}questions] N\'umero de preguntas por chunk
  \item[{-}{-}topic] ID del tema
  \item[{-}{-}academy] ID de la academia
  \item[{-}{-}skip-agent-c] Omitir validaci\'on de calidad
\end{itemize}

Utiliza \texttt{questions\_done.csv} para evitar reprocesar documentos ya
completados.

\subsection{run\_rewriter.py --- Solo Agente Z}

Ejecuta \'unicamente la fase de fragmentaci\'on. \'Util para preparar chunks antes de
lanzar la generaci\'on.

\begin{tcolorbox}[colback=bglight, colframe=bordercolor, boxrule=0.5pt, arc=2pt]
{\small\ttfamily
python run\_rewriter.py \textbackslash\\
\hspace*{1em}--auto-topic \textbackslash\\
\hspace*{1em}--limit 5 \textbackslash\\
\hspace*{1em}--limit-chunks 10 \textbackslash\\
\hspace*{1em}--force ~~\# re-procesar existentes
}
\end{tcolorbox}

\textbf{Salida}: Archivos JSON en \texttt{input\_docs/rewritten/} que pueden
ser consumidos por \texttt{run\_bcde\_pipeline.py}.

\subsection{run\_bcde\_pipeline.py --- Pipeline sin Chunking}

El punto de entrada m\'as flexible para iteraci\'on r\'apida. Consume los chunks
cacheados por el Agente~Z y ejecuta B, C, D, E.

\begin{tcolorbox}[colback=bglight, colframe=bordercolor, boxrule=0.5pt, arc=2pt]
{\small\ttfamily
python run\_bcde\_pipeline.py \textbackslash\\
\hspace*{1em}--doc "Tema-18" \textbackslash\\
\hspace*{1em}--questions-per-chunk 5 \textbackslash\\
\hspace*{1em}--workers 4 \textbackslash\\
\hspace*{1em}--skip-agent-c
}
\end{tcolorbox}

\textbf{Caracter\'isticas clave}:
\begin{itemize}[nosep,leftmargin=*]
  \item Procesamiento paralelo con \texttt{ThreadPoolExecutor}
  \item Cache de uso de chunks para evitar reprocesamiento
  \item Hilo escritor basado en cola para escrituras at\'omicas a BD
  \item Configurable: n\'umero de workers, preguntas por chunk
\end{itemize}

\subsection{extract\_and\_generate\_pdf.py --- Solo PDFs}

Utilidad para generar PDFs a partir de preguntas ya existentes en la base
de datos. No ejecuta ning\'un agente generativo.

\begin{enumerate}[nosep,leftmargin=*]

  \item Indexa en FAISS para b\'usqueda eficiente
  \item Para cada chunk primario, recupera m\'aximo 3 relacionados
  \item Eval\'ua suficiencia del contexto combinado
  \item Retorna \texttt{EnrichedContext} al Agente~B
\end{enumerate}

\section{M\'etricas y Trazabilidad}
% ============================================================================

El sistema mantiene trazabilidad completa en cada pregunta generada:

\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Origen}: documento fuente, p\'agina, chunk ID
  \item \textbf{Calidad}: scores de faithfulness y relevancy
  \item \textbf{Generaci\'on}: modelo LLM, prompt usado, m\'etodo
  \item \textbf{Deduplicaci\'on}: flag de duplicado, ID del original
  \item \textbf{T\'ecnicas}: array JSON de t\'ecnicas de distracci\'on usadas
  \item \textbf{Reintentos}: contador de intentos, flag de revisi\'on manual
\end{itemize}

Esta trazabilidad permite:
\begin{enumerate}[nosep,leftmargin=*]
  \item Auditar la calidad de las preguntas generadas
  \item Identificar chunks que producen preguntas de baja calidad
  \item Optimizar prompts bas\'andose en m\'etricas hist\'oricas
  \item Migrar a Supabase manteniendo toda la metadata
\end{enumerate}

\section{Conclusiones}
% ============================================================================

El sistema multi-agente presentado ofrece una soluci\'on completa y extensible
para la generaci\'on autom\'atica de preguntas tipo test. La separaci\'on en cinco
agentes especializados permite:

\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Modularidad}: Cada agente puede evolucionar independientemente
  \item \textbf{Calidad}: El sistema dual de evaluaci\'on con retry autom\'atico
        garantiza est\'andares m\'inimos
  \item \textbf{Escalabilidad}: El procesamiento paralelo y la cach\'e de chunks
        optimizan el throughput
  \item \textbf{Trazabilidad}: Cada pregunta conserva su historial completo
  \item \textbf{Flexibilidad}: Cuatro puntos de entrada para diferentes
        casos de uso
\end{itemize}

El dise\~no del esquema de base de datos compatible con Supabase facilita la
futura migraci\'on a infraestructura cloud sin p\'erdida de datos ni metadata.
